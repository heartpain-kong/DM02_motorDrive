## 功能与注意事项
1. 关于RS485配置
   虽然官方生成代码中已经有HAL_RS485Ex_Init(本代码中usart.c中198和242)的初始化但是我们发现在cubeMX上生成的高低电平的切换时间刻默认为0,这对于高速频率来说不太友好。
   ```
   HAL_StatusTypeDef HAL_RS485Ex_Init(UART_HandleTypeDef *huart, uint32_t Polarity, uint32_t AssertionTime,uint32_t DeassertionTime)
   ```
   在rs485.c中我时间刻给2可以自行通过自己尝试来修改。因为有cubeMX对RS485支持所以我们不需要向大疆C板一样手动切换电平,所以在达妙02开发板中只需要进行DMA的发送和接收就可以了(我一开始也是这样想的),但是因为cache原因我们对于数据操作会变得很严谨,DMA不允许访问 0x20000000 -  0x24000000 这个区间如果对此还有疑惑可以看
   
   >https://blog.csdn.net/XIanhl/article/details/154612804?sharetype=blog&shareId=154612804&sharerefer=APP&sharesource=qq_65272145&sharefrom=qq
   
   这篇文章。当然我DMA为单缓冲区,后续也会更新双缓冲区的代码,区别就为单缓冲区需要用接收返回中断,双缓冲区需要修改为空闲中断,需要注意配置双缓冲区需要关闭DMA。
   ```
   HAL_StatusTypeDef HAL_DMAEx_MultiBufferStart(DMA_HandleTypeDef *hdma,  uint32_t SrcAddress, uint32_t DstAddress, uint32_t SecondMemAddress,  uint32_t DataLength)
   ```
   HAL库已经提供了DMA双缓冲区的初始化。
<br>

2. 关于宇树
   宇树的GO-M8010-6电机是单编码器并且减速比为6.33,这意味着上电需要保证一个位置并且把此位置编码器返回值作为零点(通过代码实现而不是电机自带)。宇树在实际使用中会出现xt30 2+2线断电源这会使零点位置改变(电机硬件上的)因为圈数会清除。使用中需要注意。
<br>

3. 关于CAN_FD
   我选择了480Mhz主频120Mhz分频到CAN_FD上,所以cubeMX配置我延用了麻神的,这没有什么好说的。
<br>

4. 关于sct配置
 如果开启了Cache后需要用sct配置地址区创建一些需要访问的变量的时候对于所在地址区固定,不然就会和DMA一样访问不到。以下为sct配置,参考王草凡大大的文章。

>https://zhuanlan.zhihu.com/p/4218673539

 ```
; *************************************************************
; *** Scatter-Loading Description File generated by uVision ***
; *************************************************************

LR_IROM1 0x08000000 0x00100000  {    ; load region size_region
  ER_IROM1 0x08000000 0x00100000  {  ; load address = execution address
   *.o (RESET, +First)
   *(InRoot$$Sections)
   .ANY (+RO)
   .ANY (+XO)
  }
 
  RW_IRAM1 0x20000000 0x00020000  {  ; RW data
   .ANY (+RW +ZI)
  }
 
 RW_IRAM2 0x24000000 0x00020000  {
   .ANY (.AXI_SRAM)
  }

 RW_IRAM3 0x30000000 0x00004000  {
   .ANY (.SRAM1)
  }

 RW_IRAM4 0x30004000 0x00004000  {
   .ANY (.SRAM2)
  }

 RW_IRAM5 0x38000000 0x00004000  {
   .ANY (.SRAM4)
  }

}
```
但是要注意要在魔术棒/Target中IRAM2的地址改为start:0x0 Size:0x10000
